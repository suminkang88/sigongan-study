## Git이란?

버전 관리 툴. 로컬에서 작업한 결과물을 원격 저장소에 보관하고, 변경사항을 "버전"으로 기록한다.

### Git이 없으면?
- 누가 어떤 코드를 작성했는지 알 수 없음
- 변경사항마다 백업본을 만들어 수동으로 공유해야 함
- 여러 명이 같은 파일을 수정하면 작업물이 꼬임
- 기획 변경 시 어떤 코드를 살리고 버려야 할지 추적 불가

### Git을 쓰면?
- 누가, 언제, 무엇을, 어디서, 왜 변경했는지 기록됨
- 버전 간 차이점 비교 가능
- 충돌 발생 위치 파악 가능
- 특정 버전으로 롤백 가능

## GitHub란?

Git이 로컬에서 버전을 관리하는 도구라면, GitHub는 그 Git 저장소를 클라우드에서 호스팅해주는 웹 서비스다.

쉽게 말해:
- **Git** = 버전 관리 시스템 (도구 자체)
- **GitHub** = Git 저장소를 올려두는 플랫폼 (저장 공간 + 협업 공간)

### GitHub의 주요 기능
- **원격 저장소** — 로컬 Git 저장소를 클라우드에 백업 및 공유
- **Pull Request** — 코드 변경사항을 팀원이 리뷰하고 병합하는 흐름
- **Issue 트래킹** — 버그, 기능 요청 등을 티켓으로 관리
- **오픈소스 생태계** — 전 세계 개발자들의 코드를 보고, 기여할 수 있음

> GitHub를 "개발자들의 SNS"라고 부르기도 한다. 내가 어떤 프로젝트를 얼마나 꾸준히 작업했는지 한눈에 보이기 때문.

## 핵심 명령어

| 명령어 | 설명 |
|--------|------|
| `git add` | 변경사항을 스테이징 영역에 추가 |
| `git commit` | 스테이징된 변경사항을 버전으로 기록 |
| `git push` | 로컬 커밋을 원격 저장소에 업로드 |
| `git pull` | 원격 저장소의 변경사항을 로컬로 가져옴 |
| `git status` | 현재 작업 상태 확인 |
| `git clone` | 원격 저장소를 로컬로 복제 |
